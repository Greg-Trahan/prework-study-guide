Prework Study Guide

HTML

  • The beggining of each HTML file must contain '!DOCTYPE html' and 'html lang="en"'.
  • The head element contains information about the webpage.
    • 'meta charset="UTF-8"'
    • 'meta http-equiv="X-UA-Compatible" content="IE=edge"' - This may not be needed.
    • 'meta name="viewport" content="width=device-width, initial-scale=1.0"''
    • 'link rel="stylesheet" href="./path/to/file.css"' - This will link a CSS sheet to this HTML page
    • '>title *** /title<'
  • The body element represents the visible content shown to the user.
    • 'header id="top"'
    • 'section class="card" id="*section title*'
    • Adding a 'class="name"' to a section allows you to isolate that segment in CSS
    • 'h1-h6'
    • 'ul, ol with id elements'

CSS

  • Each CSS element has two parts
    • Selector - Defines the element or attribute to appply the decloration to
    • Declaration
      • Property - what we are affecting (ie. color, font size)
      • Value - what we are setting the property to
  • CSS Elements
    • CSS elements are formated as 'Selector, selector2 {property: value; property2: value2}'
    • * - A selector which will apply to all elements on the page
    • Margin - border around the page
    • Padding - border around the content
    • Text align
    • Width
    • Height
    • Color
    • Background color
    • Border
    • Box-shadow
    • '.card {' allows you to specify a particular class in the HTML file to modify
    • Display
      • Block - will obey width and height properties. Padding, margin and border will cause other elements to be pushed away from itself.
      • Inline - Will not apply width and height - Vertical padding, margins, and borders will apply but will not cause other inline boxes to move away from the box. Horizontal padding, margins, and borders will apply and will cause other inline boxes to move away from the box.
      • Flex - The element will still use the outer display type block but this changes the inner display type to flex. Any direct children of this box will become flex items and behave according to the Flexbox specification.

Git

  • Some commands are not git, but for navigating the Terminal/GitBash
    • ls: command to view all files in current folder
    • cd folder-name: command to move into a new folder
    • ls ..: command to view files in parent folder
    • cd ..: command to move into parent folder
    • pwd: check current directory
    • mkdir: create a new directory
    • touch file-name: creatue a new file
    • start file-name: open selected file
  • Other commands are specific for git. These all must begin with 'git'
    • git status: checks what branch we are currently on
    • git checkout branch-name: switch to listed branch
    • git checkout -b branch-name: creates a new branch and switches to it
    • git pull origin branch-name: updates files to the most recent version of that branch
      • pull: retrieve branch modifications to local environment
      • origin: the source of the pull is GitHub repo
    • git add -A: add all of the changes made to files to the branch staging area
    • git commit -m "details here": commit staging area files to local repo with an imperative command description
    • git push origin branch-name: push the local branch changes to GitHub repo
